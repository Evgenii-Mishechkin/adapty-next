# История запросов/ответов

Ниже — последовательность задач, выполненных в рамках разработки проекта.  
Формат: **Prompt → Expected outcome**.

---

## 1) Структура проекта (дерево)

**Prompt**  
Собери структуру проекта под Next.js App Router: папки и файлы. Реализация не нужна — только дерево директорий и файлов.

**Expected outcome**  
- Показано дерево `app/(marketing)`, `components/marketing`, `components/ui`, `lib`, `types` с нужными `page.tsx` и `layout.tsx`.

---

## 2) Создать директории по дереву

**Prompt**  
Создай все директории в проекте согласно утверждённой структуре.

**Expected outcome**  
- Все папки созданы.  
- Структура соответствует согласованному дереву.

---

## 3) Исправление: всё должно быть внутри `src/`

**Prompt**  
Вся структура должна находиться внутри `src/`. Перенеси или создай каталоги так, чтобы использовались пути `src/app`, `src/components`, `src/lib`, `src/types`.

**Expected outcome**  
- Вся структура расположена внутри `src/…`.

---

## 4) Удалить лишнее, созданное вне `src/`

**Prompt**  
Удали все каталоги, которые были ошибочно созданы вне `src/` (дубликаты `app/`, `components/`, `lib/`, `types`). Оставь только структуру внутри `src/`.

**Expected outcome**  
- В корне проекта нет дублирующихся директорий.  
- Вся структура находится в `src/`.

---

## 5) Создать все файлы внутри созданных директорий

**Prompt**  
Создай все файлы из утверждённого дерева внутри `src/`. На этом этапе допустимы пустые файлы.

**Expected outcome**  
- Все `*.tsx` и `*.ts` файлы существуют по своим путям.

---

## 6) Начинаем верстать страницу блоками

**Prompt**  
Дальше работаем по скриншотам секций. Для каждой секции делай отдельный компонент и подключай его на главной странице. Указывай, какие файлы были изменены.

**Expected outcome**  
- Для каждой секции есть отдельный компонент.  
- Компоненты подключены на главной странице.

---

## 7) Hero-секция (desktop 1920)

**Prompt**  
Сверстай hero-секцию под 1920px по макету: бейдж «Гайд…», крупный градиентный заголовок, описание, поле email с CTA и ссылка «Запись на демо».  
Справа — визуальный блок (dashboard + phone).  
Используй Tailwind. Без клиентской интерактивности.

**Expected outcome**  
- `HeroSection` соответствует макету (desktop-first).  
- Адаптив работает корректно.  
- Нет ошибок Next.js, связанных с server components.

---

## 8) Исправления сборки: дубли и кодировка

**Prompt**  
Исправь ошибки сборки:  
- убери дублирующиеся определения компонентов и layout’ов;  
- приведи строки и metadata к корректной UTF-8 кодировке.

**Expected outcome**  
- Приложение собирается без ошибок.  
- Тексты отображаются корректно.

---

## 9) Hero: изображения из `public` + логотипы

**Prompt**  
Ассеты уже добавлены в `public`. В hero подключи реальные изображения для dashboard и phone и добавь под секцией строку с логотипами, как в макете.

**Expected outcome**  
- Используется `next/image` и файлы из `public`.  
- Под hero отображается блок с логотипами.

---

## 10) Hero: текст не должен пересекаться с медиа

**Prompt**  
Исправь адаптив так, чтобы текст hero-секции не заходил под правый медиа-блок на промежуточных ширинах экрана. Desktop-композицию не ломать.

**Expected outcome**  
- На tablet/laptop нет пересечений текста и изображений.

---

**Expected outcome**

## 18) Финальная проверка

**Prompt**  
Проведи финальную проверку страницы:  
- нет дублирующихся компонентов и layout’ов;  
- нет server/client ошибок;  
- все секции подключены в нужном порядке;  
- футер отображается.

Кратко опиши, какие файлы менялись, и что проверить визуально.

**Expected outcome**  
- Проект собирается без ошибок.  
- Страница визуально соответствует макету.
